# 智评 ZhiPing — 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品定位
智评是一款面向英语学习者的AI写作智能批改工具。用户上传英语作文后，系统自动进行多维度评分（词汇、语法、结构、内容）并生成个性化、可操作的写作反馈。

### 1.2 核心价值主张
- **效率提升**：将批改一篇作文从教师的15分钟缩短至5秒
- **一致性**：消除不同教师评分标准不一致的问题
- **即时性**：学生可在任何时间获得即时反馈，加速学习循环
- **个性化**：根据学生水平调整反馈的深度和语言复杂度

### 1.3 目标用户
| 用户角色 | 痛点 | 使用场景 |
|----------|------|----------|
| 大学英语教师 | 批改60篇作文耗时15小时 | 批量预批改，教师复审后发给学生 |
| 雅思考生 | 写作练习无即时反馈 | 自主练习时获得AI批改，了解改进方向 |
| 国际学校 | 外教成本高，批改质量参差 | 辅助外教批改，统一评分标准 |
| 语言培训学校 | 续费率依赖教学效果 | 作为增值服务提升用户粘性 |

## 2. 市场分析

### 2.1 市场规模
- 中国在线英语教育市场：2024年约800亿元人民币
- AI教育市场年增长率：>25%（艾瑞咨询）
- 全球AI写作评估市场：预计2030年达到40亿美元（Grand View Research）

### 2.2 竞品分析
| 产品 | 优势 | 劣势 | 差异化机会 |
|------|------|------|-----------|
| Grammarly | 语法纠正精准，插件生态完善 | 不适合二语学习者，反馈偏语言而非内容 | 二语学习者专用反馈 |
| 批改网 | 中国市场占有率高 | 评分粗糙，反馈模板化 | LLM驱动的个性化反馈 |
| Write & Improve (剑桥) | 专业评分标准 | 反馈不够详细 | 多维度的详细反馈 |
| ChatGPT | 灵活通用 | 无评分标准，不专门为写作评估设计 | 基于专业评分量规的系统性评估 |

### 2.3 我们的优势
1. **学术背景**：基于语言学专业知识和ELLIPSE语料库验证
2. **双模评分**：特征工程+LLM，兼顾速度与精度
3. **数据隐私**：支持本地模型部署，学生数据不出校园
4. **可解释性**：不仅给分数，还给出分数背后的具体原因

## 3. 产品架构

### 3.1 技术架构
```
┌─────────────────────────────────────────────────────┐
│                    前端 (Gradio / React)              │
├─────────────────────────────────────────────────────┤
│                   API Layer (FastAPI)                │
├──────────────────┬──────────────────────────────────┤
│  特征评分引擎    │         LLM评分引擎               │
│  (Linear Reg.)   │  (DeepSeek / Qwen-1.5B)          │
├──────────────────┴──────────────────────────────────┤
│              反馈生成引擎                             │
│  (结构化Prompt + 温度控制 + 水平适配)                │
├─────────────────────────────────────────────────────┤
│              数据层 (评分结果 / 反馈 / 学生档案)      │
└─────────────────────────────────────────────────────┘
```

### 3.2 核心功能模块
| 模块 | 功能描述 | 优先级 |
|------|----------|--------|
| 作文上传 | 支持粘贴/文件上传/图片OCR | P0 |
| AI评分 | 1.0-5.0综合评分 | P0 |
| 多维分析 | 词汇/语法/结构/内容4维度雷达图 | P0 |
| 详细反馈 | 引用错误原文+解释+改进版本 | P0 |
| 批量批改 | 教师一次上传30-60篇作文 | P1 |
| 进步追踪 | 学生历史评分趋势图 | P1 |
| 班级管理 | 教师创建班级、查看学生数据 | P2 |
| 隐私模式 | 本地LLM部署，数据不出校园 | P2 |

## 4. 商业化路径

### 4.1 盈利模式
| 模式 | 价格 | 目标客户 |
|------|------|----------|
| C端订阅 | ¥29/月，¥199/年 | 雅思考生、自学者 |
| B端SaaS | ¥5000-20000/校·年 | 国际学校、培训机构 |
| API按量 | ¥0.05/次批改 | 教育平台集成 |
| 企业定制 | 项目制定价 | 大型教育集团 |

### 4.2 增长策略
- **PLG（产品驱动增长）**：免费试用5篇作文 → 付费转化
- **KOL合作**：与雅思名师合作推广
- **开放API**：允许教育平台接入

## 5. 成功指标

### 5.1 北极星指标
**月活跃批改次数**（反映产品核心价值被使用的频率）

### 5.2 关键指标
| 指标 | 目标值 | 衡量方式 |
|------|--------|----------|
| 评分准确率 (Pearson r) | > 0.65 | 与人工评分对比 |
| 反馈准确率 | > 90% | 专家抽检评估 |
| 用户留存率 (月) | > 60% | 次月仍在使用的比例 |
| 付费转化率 | > 8% | 免费→付费转化 |
| NPS | > 50 | 用户净推荐值 |
